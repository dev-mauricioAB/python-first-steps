# python-fisrt-steps

A personal repository to document the first stage of a Python study journey focused on backend growth, practical exercises, and hands-on experimentation. This project was created to strengthen core Python knowledge and build a solid foundation for future work with FastAPI, pandas, and AI agents.

## About this repository

This repository is part of a learning roadmap designed for a senior frontend engineer moving deeper into backend development with Python.

The main goals of this study are:

- Understand the basic and main Python concepts used in backend development.
- Become more comfortable reading and writing Python in real project contexts.
- Learn how Python is used to iterate, transform, and process data.
- Build a foundation for working with FastAPI, pandas, and LangGraph.
- Prepare for understanding and contributing to Python-based agent workflows in real projects.

## Why this repository exists

This repository is not intended to be a polished production project. Its purpose is to publicly track learning, practice consistency, and show the evolution of Python skills through small exercises, notes, and practical code examples.

It also works as a study log that highlights:

- Daily or weekly Python practice.
- Core syntax and language fundamentals.
- Backend-oriented exercises.
- Data manipulation practice.
- Preparation for API development and AI-agent orchestration.

## Study approach

The study strategy used in this repository is based on learning Python through practical backend use cases instead of studying it only as an academic language.

That means the focus is on:

- Writing small scripts.
- Reading and transforming JSON-like data.
- Practicing functions and collections.
- Handling files and exceptions.
- Organizing code into modules.
- Building a mental bridge from TypeScript backend concepts into Python.

## Main topics studied

### 1. Variables and data types

Learned how Python represents values such as strings, integers, booleans, lists, and dictionaries, and how these structures are commonly used to model payloads and structured data.

### 2. Functions

Practiced writing reusable functions, defining parameters, returning values, and using type hints to improve clarity.

### 3. Control flow

Studied `if`, `elif`, `else`, `for`, and `while` to control program execution and process data conditionally.

### 4. Collections

Worked with lists, dictionaries, tuples, and sets, including simple transformations and filtering logic that are common in backend tasks.

### 5. Exceptions

Practiced using `try`, `except`, `else`, and `finally` to handle runtime problems safely.

### 6. File I/O

Learned how to open, read, and write files, especially JSON files, which are very common in backend workflows.

### 7. Modules

Started organizing code into separate files and importing reusable functions to simulate real project structure.

## Practice exercises

Examples of exercises developed in this repository:

- Declare variables using Python type hints.
- Build functions with parameters and default values.
- Filter active users from a list of dictionaries.
- Transform data using list comprehensions.
- Manipulate dictionaries and update values.
- Handle errors such as invalid input and division by zero.
- Parse JSON strings and files.
- Split logic into modules.
- Flatten nested lists from structured objects.
- Process API-like responses and return summarized results.

## Study plan

The full study plan for this journey is organized into practical phases.

### Phase 1 — Core Python for backend thinking

Focus on:

- Variables, functions, control flow, and collections.
- Exceptions and file handling.
- Modules and project organization.
- Writing small backend-oriented scripts.

Expected outcome:

- Read and write basic Python confidently.
- Understand the syntax and core programming model.
- Create small programs that manipulate structured data.

### Phase 2 — Pythonic data handling

Focus on:

- List and dictionary comprehensions.
- Iterators and generators.
- Type hints.
- Structured data transformation.
- Dataclasses and validation-oriented thinking.

Expected outcome:

- Write more expressive Python code.
- Transform nested and structured data more naturally.
- Improve readability and maintainability.

### Phase 3 — FastAPI fundamentals

Focus on:

- Creating APIs with FastAPI.
- Routes, request bodies, query parameters, and validation.
- Response models and error handling.
- Async endpoints and project structure.

Expected outcome:

- Build small APIs with Python.
- Understand Python backend patterns in web services.
- Connect existing backend knowledge to FastAPI conventions.

### Phase 4 — pandas for practical data work

Focus on:

- DataFrame basics.
- Reading CSV, Excel, and JSON files.
- Filtering, grouping, cleaning, and joining data.
- Returning transformed data in useful formats.

Expected outcome:

- Handle structured datasets more efficiently.
- Understand how Python is used to iterate and process data in real scenarios.
- Prepare for internal tools and data-driven APIs.

### Phase 5 — LangGraph and AI agent workflows

Focus on:

- State-based workflows.
- Nodes and edges.
- Tool-calling orchestration.
- Building simple agent flows.
- Understanding the structure of Python-based AI agents.

Expected outcome:

- Understand how Python can be used to coordinate AI agents.
- Build a mental model for agent orchestration.
- Be more prepared to work with Python-based AI systems in professional projects.

## Weekly study rhythm

A practical weekly rhythm used for this plan:

1. Study one concept from documentation or a focused tutorial.
2. Rebuild the concept from memory.
3. Apply it to a small backend-oriented example.
4. Compare the Python approach with TypeScript when helpful.
5. Finish with one small working exercise or script.

## Long-term goal

The long-term goal of this repository is to document the transition from Python basics to practical backend and AI-agent use cases.

This includes progressing from:

- simple scripts,
- to data transformation,
- to API development,
- to more advanced Python workflows used in modern backend and AI projects.
